# Security

Do **not** report security vulnerabilities through public GitHub issues.

## Reporting a vulnerability

1. **GitHub** — use the repository **Security** tab and **Report a vulnerability** (private advisory), if enabled for this repo.
2. **Email** — if you cannot use advisories, contact the maintainers privately at **i@borodatych.ru** with a clear subject (e.g. `VibeIDE security`).

Please include enough detail to reproduce or assess impact (versions, steps, impact). We will coordinate disclosure after a fix.

Upstream VS Code security processes and Microsoft guidance remain relevant for code inherited from `microsoft/vscode`; see [https://aka.ms/SECURITY.md](https://aka.ms/SECURITY.md) for Microsoft’s policy where it applies.
