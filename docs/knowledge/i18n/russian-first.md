# Russian-first: все пользовательские тексты — на русском

← [Knowledge Index](../README.md)

---

## [правило] Любой user-facing текст пишется на русском прямо в исходнике

**Контекст:** VibeIDE — русскоязычный продукт. Исторически часть строк добавлялась
по-английски «как в апстриме» и переводилась потом массовыми зачистками (2026-06-07:
~390 строк в 60 файлах за один заход). Это дорого и оставляет дыры — пользователь
регулярно ловил английские баннеры (пример: «Stream stalled — no tokens received…»).

**Суть:** русский текст пишется СРАЗУ при добавлении строки, во всех слоях:
- default-строка `nls.localize(key, 'Русский текст…')` — в т.ч. описания настроек,
  заголовки команд, тосты, диалоги;
- строки React-чата (`vibeSettingsRu.ts` и инлайн в компонентах);
- сырые строки ошибок, уходящие пользователю из main-процесса (например, тексты
  watchdog'ов в `aiSdkAdapter`);
- провайдерские английские ошибки переводит `common/providerErrorTranslator.ts` —
  новые типовые семейства добавлять туда (со своим тестом).

**Исключения (английский допустим только здесь):**
- технически невозможен русский: HTTP `statusText` и прочие ByteString/Latin-1-поля —
  не-ASCII там РОНЯЕТ конструктор `Response` (инцидент 2026-06-07, em-dash, код 8212);
- программные идентификаторы: command id, конфиг-ключи, имена тулов, JSON-поля;
- имена продуктов/брендов (VibeIDE, GitHub, Ollama…);
- текст, адресованный МОДЕЛИ, а не пользователю (system prompt, corrective-вставки —
  по ситуации; у нас corrective-подпины русские осознанно);
- комментарии в коде — всегда английские (правило CLAUDE.md).

**Применение:** на code review любая новая англоязычная строка в `localize()` /
React / notify / error-path — блокер, переводить до коммита. Проверка хвостов:
`grep "localize\((?:'[^']+'),\s*'[A-Z][a-z]+ "` по `contrib/vibeide`.
