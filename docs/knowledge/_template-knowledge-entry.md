## [роль] [Короткое описание]

> Role tags: `[инструмент]` / `[баг]` / `[инцидент]` / `[правило]` / `[архитектура]` / `[квирк]`. Выбрать одну подходящую.

**Контекст:** что за ситуация / откуда взялось / какой код / какой инцидент. 1-3 предложения. Не дублировать общеизвестное.

**Суть:** ключевые факты с минимумом воды. Можно code-блок'ом, таблицей, bullet list'ом. Цель — через 6 месяцев читатель восстановит причину решения.

**Применение:** что делать в проекте — конкретно, операционно. Если правило — формулировка одной строкой. Если quirk — что отметить при code review / debugging.

**Антипаттерны** (опционально): что НЕ делать. Только если был соблазн неправильного решения и важно зафиксировать почему.

**Связано** (опционально): cross-references на другие файлы в knowledge базе через `[[wikilink]]` или relative `[text](../topic/file.md)`.

## Convention reminders

- Заголовок: `# [Заголовок темы]` — однострочный, в setItem стиле.
- Role tag в квадратных скобках перед текстовым заголовком: `## [инцидент] 2026-MM-DD — Renderer OOM via PlanResume`.
- Дата для инцидентов: ISO `YYYY-MM-DD`.
- Цитаты кода: `\`<inline>\`` для коротких / fenced block для multi-line.
- Ссылки на code: `[file.ts:42](../../../src/.../file.ts#L42)` — full relative path + line.
- Имена tool / setting — `\`vibeide.foo.bar\`` inline code.
- Tone: рабочие записи, не маркетинг. Цель — фиксация знания, не «впечатлить».
