# edit_file — выравнивание отступа при толерантном матче

**Инцидент (Promed, экспорт сессии 2026-06-19).** Агент 15+ ходов бодался с `edit_file` и словил anti-loop брейкер (9 повторов), пытаясь вставить блок в файл с 8-пробельным отступом. Он решил, что инструмент «стабильно режет первую строку new_string», и перебирал обходы (плейсхолдеры, two-step, rewrite_file, скрипты).

## Корень

Бага-«trim» **не было**. Механика `edit_file`:

1. `findTextInCode` ищет `old_string` точным `indexOf`; при промахе — построчный толерантный fallback (`findLinesTolerant`), который **игнорирует отступ** и возвращает диапазон ЦЕЛЫХ строк.
2. `_instantlyApplySRBlocks` заменяет эти строки на `new_string` **дословно**. Отступ совпавшей строки в файле **отбрасывается**.

Модель прислала `old_string = "ctrl.mask(...)"` без 8 пробелов и `new_string`, у которого первая строка без отступа, остальные — с 8. Движок заменил всю строку `        ctrl.mask(...)` блоком, чья первая строка идёт с колонки 0 → рваная «ёлочка». Модель неверно диагностировала это как «trim первой строки» и ушла в спираль.

## Фикс (v1.2.4)

Три рычага, дополняющие друг друга:

1. **Авто-выравнивание на толерантном матче** — `common/helpers/reindentSearchReplace.ts` (`alignReplacementIndentation`). Срабатывает ТОЛЬКО когда `indexOf` промахнулся (т.е. отступ реально отличался); точный матч остаётся байт-в-байт. Различает два кейса:
   - **консистентный сдвиг** (всё тело дедентнуто относительно файла) → равномерный сдвиг всех строк на дельту;
   - **потеря отступа только у первой строки** (тело уже на уровне файла, первая строка — выброс) → правится лишь первая строка, тело не трогается.
   `findTextInCode` репортит `matchModeRef.exact`. Покрыто юнит-тестами `test/common/reindentSearchReplace.test.ts`.
2. **Информативная нота в результате тула** — при срабатывании выравнивания `edit_file` возвращает не «success» молча, а пометку: «отступ не совпал, я выровнял; проверь diff, иначе пришли точный отступ КАЖДОЙ строки или используй rewrite_file; инструмент вставляет дословно». Превращает молчаливое рассогласование в самокорректирующийся сигнал → фикс за 1 ход вместо спирали. Anti-loop брейкер остаётся страховкой.
3. **Уточнение описания тула** (`prompt/tools/edit_file.ts`) — явно: вставка ДОСЛОВНАЯ, инструмент НЕ переотступает, сохраняй точный отступ каждой строки, включая первую.

## Урок

Когда инструмент применяет правку дословно, а матч толерантен к отступу, рассогласование отступа **молчаливо** и легко принимается моделью за «баг инструмента». Лечится связкой: авто-выравнивание частого кейса + явная нота на остаток + честное описание семантики. Чистая строковая логика выравнивания вынесена в `common/` и тестируется из `test/common/`.
